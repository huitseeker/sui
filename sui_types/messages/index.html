A transaction signed by a client, optionally signed by an authority (depending on S). S indicates the authority signing state. It can be either empty or signed. We make the authority signature templated so that TransactionEnvelope<S> can be used universally in the transactions storage in SuiDataStore, shared by both authorities and non-authorities: authorities store signed transactions, while non-authorities store unsigned transactions.
